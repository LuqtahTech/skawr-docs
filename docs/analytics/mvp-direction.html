MVP Direction

The shortest path from zero to "first paying MENA marketplace customer."

What we copy from Amplitude (table stakes)

Ship or lose. If any of these are missing on day 1, we look like a toy.

  • SDKs for web (JS), iOS, Android, Node. Server-side ingest.
  • Autocapture on web (page views, clicks, form submits) so Day 1 isn't empty.
  • Event tracking with properties + user properties.
  • Funnels (multi-step, with breakdown by property).
  • Retention curves.
  • User journeys / pathfinder view.
  • Dashboards + saved views.
  • Demo project pre-loaded on signup.
  • Time-to-first-event under 5 minutes.

What we beat Amplitude on — the four pillars at MVP

Pillar 1 — Smart insights (the daily-use UX)

  • Insight feed is the home screen. Not a sidebar, not a tab — the first thing you see.
  • MVP detection types:
    • Anomalies on top metrics (DAU, signups, key event counts) using simple statistical baselines.
    • Funnel drop-off detection — find steps where conversion is significantly worse than expected.
    • Driver attribution — for any anomaly, which segment / property explains the change ("Android users on Chrome 119" / "users from organic search").
  • Every insight links to the supporting chart. Click to see the math. No black box.
  • No chatbot. Insights are explicit, scoped, and verifiable.

Defer to Phase 2: predictive insights, forecasting, churn prediction.

Pillar 2 — Embeddings-powered behavioral clustering

  • MVP scope: narrow but real. One auto-discovered cohort surfaced per project per week.
  • Pipeline: encode each user's last-30-days event sequence as a vector → cluster (HDBSCAN or similar) → score clusters by uplift on a key metric (conversion, retention) → surface the top-uplift cluster as an insight.
  • The insight names the cluster ("4,200 users who view → favorite → return-within-2-days") and quantifies its conversion advantage vs the baseline.
  • Reuses Fireworks embeddings stack already operating in skawr-indexer. Real moat — almost nobody in product analytics does this.

Defer to Phase 2: real-time cluster assignment, cohort comparison, predictive cohort flow.

Pillar 3 — MENA stack (the moat)

  • Arabic UI, RTL-first. Designed RTL, not bolted on.
  • Hijri calendar option alongside Gregorian.
  • Time periods are Ramadan / Eid aware (Ramadan-week comparisons are flagged, not silently regressed).
  • Data residency: AWS me-south-1 (Bahrain) primary; me-central-1 (UAE) for customers requiring it.
  • PDPL (Saudi) compliance baked in from day 1.
  • Local pricing: SAR / AED first-class, USD secondary.
  • Local payment: Mada, Tabby, Tamara alongside cards.
  • Arabic docs, Arabic onboarding emails, Arabic in-app support.

Pillar 4 — Vertical templates (MVP: marketplaces only)

Why marketplaces first

  • We already know the domain (Skawr-Marketplace, skawr-scraper has scraped Haraj / Dubizzle / OpenSooq / Mstaml).
  • Saudi / UAE marketplace startups are a warm first-customer pool.
  • Marketplace funnels are well-defined and high-value (deal completion = revenue).

What ships in the marketplace template

Pre-defined event taxonomy:

listing_view listing_favorite seller_contact message_sent message_received phone_revealed deal_closed listing_created listing_renewed listing_expired report_filed

Pre-built funnels:

  • view → contact → message → close
  • create → first-view → first-contact
  • signup → first-listing → first-deal

Pre-built dashboards: supply/demand health (active listings vs active buyers); seller cohorts (by category, by tier); buyer cohorts (by intent depth); category-level conversion; chat-to-close ratio; price-band conversion.

MENA marketplace benchmarks: "Your listing-to-contact ratio is in the bottom quartile vs MENA marketplaces in your category." Benchmark dataset built from anonymized customer data over time (chicken-and-egg: seed with our own scraped data + manual industry estimates).

Defer Phase 2: fintech template (KYC, deposit, transaction, churn), edtech template (enrollment, lesson, completion).

Pricing — the hook

Pricing page headline: "Never get punished for growing."
  • Flat-rate or event-volume tiers. No MTU. No "punish you for retaining users."
  • Free tier generous enough that early-stage MENA startups can ship to production without paying.
  • Local pricing: SAR / AED first-class, USD secondary.
  • Local payment: Mada, Tabby, Tamara alongside cards.

What we explicitly defer (post-MVP)

  • Session replay (planned, but not Day 1).
  • A/B testing / Experiment.
  • Feature flags.
  • Reverse ETL / CDP features.
  • Open-source / self-host.
  • Natural-language chatbot (intentional — explicitly rejected).
  • Fintech and edtech vertical templates.
  • Predictive insights (churn forecasts, LTV prediction).

Wedge customer (who we sell to first)

RankSegmentWhy
1 Saudi / UAE marketplace startups (10k–500k MAU) Money, care about local + Arabic, hate the Amplitude bill, and the marketplace template means they're productive in minutes. First.
2 Saudi / UAE non-marketplace product startups Same buyer profile, but slower onramp without a vertical template.
3 Global SMB startups burned by MTU pricing Bigger TAM but harder differentiation (no MENA hook, no MENA-marketplace template).

Open questions (need decisions before MVP build)

  • Tech stack. Backend language / framework? Lean: Python (FastAPI) — reuse skawr-indexer DNA. Embeddings stack already operating.
  • Event store. ClickHouse for events (industry standard for analytics), or stay on OpenSearch? Lean: ClickHouse for events; Postgres for metadata; OpenSearch only if needed.
  • Pricing tiers. Flat tier prices need actual numbers. What does the SAR-denominated ladder look like, and what's included at each?
  • Insight detection v1. Anomaly + funnel drop-off + driver attribution — that's the MVP set. Do we add anything else, or hold the line?
  • Embeddings clustering cadence. Run nightly, weekly, or on-demand? Lean: weekly — matches the "one cohort per week" insight cadence.
  • Arabic content. Hire/contract a native Arabic copywriter, or start with bilingual founders translating?
  • Brand / domain. Subdomain of skawr.com? Standalone domain? Arabic-first or English-first name?
  • Marketplace benchmark seeding. How do we get to credible benchmarks before we have customers? Options: anonymized data from skawr-scraper's existing crawl; published industry estimates; manual partner data from 2–3 friendly marketplaces.

Day 1 user flow (what activation looks like)

  1. Sign up (email or Google).
  2. Wizard: workspace name → role → vertical (marketplace / general) → app type (web / mobile / both) → language (English / Arabic).
  3. SDK snippet shown — language auto-detected. Copy-paste, paste into your app.
  4. If marketplace selected: marketplace event taxonomy is pre-loaded; SDK snippet includes example calls for listing_view and seller_contact; pre-built marketplace funnel + dashboard appear in the workspace.
  5. Autocapture begins flowing pageviews / clicks within 60 seconds of paste.
  6. Demo project pre-loaded with sample marketplace data so the dashboard isn't empty.
  7. As soon as the user's own first event arrives, the banner says "real data is in" and the workspace switches.
  8. Within 24h: first auto-insight surfaces in the inbox. (Welcome insight: top event + first anomaly if any.)
  9. Within 7 days: first behavioral cluster surfaced (Pillar 2) — system says "we found a cohort of N users who behave like this and convert M× higher than baseline."
Activation metric: % of new signups who see their own first event within 5 minutes. Bar: > 60%.
Retention metric: % of week-1 signups who return in week 4 with the insight feed showing ≥3 unread insights.
