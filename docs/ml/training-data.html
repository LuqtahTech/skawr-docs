Data Quality Notes

  • 64% of products have no price (price in description text, not structured field)
  • Titles are short (2-10 words typically)
  • Descriptions range from 10 to 2000+ characters
  • Mixed Arabic/English content common
  • Categories: phones, cars, electronics, furniture, clothing, sports, books, games

🔄 Data Processing

Text Field

Each product's text field is created by combining title and description:

text = title + ". " + description

Filtering

  • Remove products with null text
  • Remove products with text shorter than 10 characters
  • Truncate text to 500 characters (memory optimization)
  • Tokenizer truncates to 256 tokens at training time

Pair Generation

Training pairs are generated on-the-fly (not pre-computed) to save memory:

Query
First 50 characters of product title, prefixed with "query: "
Positive Product (70%)
The same product's full text, prefixed with "passage: "
Negative Product (30%)
Random product from a different domain

💾 Embeddings in Supabase

Vector Columns

title_embedding1024-dim vector from title only
description_embedding1024-dim vector from description only
combined_embedding1024-dim vector from title + description
has_embeddingBoolean flag
embedding_generated_atTimestamp

Search Function

Supabase RPC function search_products_by_similarity performs cosine similarity search against the combined_embedding column.
