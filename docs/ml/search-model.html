How It Works

The model converts product text and search queries into 1024-dimensional vectors. Products with similar meaning have vectors that are close together, enabling semantic search.

🔍
User types "iPhone 15"
↓
🧠
Model encodes: "query: iPhone 15" → [0.12, -0.34, 0.56, ...] (1024 numbers)
↓
📦
Compare against pre-computed product vectors in database
↓
✅
Return products with highest cosine similarity

E5 Prefix Convention

The E5 model requires specific prefixes to work correctly:

  • Search queries: Prefix with "query: "
  • Product documents: Prefix with "passage: "

Without these prefixes, the model treats all text as generic and cannot distinguish between "searching for iPhones" and "a document about iPhones."

🔧 LoRA Fine-tuning

Instead of training all 560M parameters (requires 20GB+ VRAM), LoRA freezes the base model and trains small adapter layers inserted into the attention mechanism.

LoRA Rank (r)16
Alpha32
Target Modulesquery, key, value, dense
Dropout0.1
VRAM Usage~6-8 GB (fits on RTX 4070 Ti)

After training, the LoRA adapters are merged back into the base model. The final model is the same size as the original E5 but with weights tuned for SKAWR marketplace search.

Epochs3
Batch Size8
Learning Rate2e-4
OptimizerAdamW
SchedulerLinear warmup (10%)
PrecisionMixed FP16
Loss FunctionInfoNCE Contrastive
CheckpointsEvery 1000 batches + per epoch

Loss Function

InfoNCE contrastive loss teaches the model that a query should match its correct product more than any other product in the batch. Temperature scaling (0.05) sharpens the similarity scores.

Training Pairs

Generated on-the-fly each batch:

70% Positive Pairs
Query from product title, paired with the full product text
30% Negative Pairs
Query paired with a random product from a different domain

🚀 Usage

Python

from transformers import AutoTokenizer, AutoModel
import torch, torch.nn.functional as F

model_path = "models/checkpoints/e5-lora/merged"
tokenizer = AutoTokenizer.from_pretrained(model_path)
model = AutoModel.from_pretrained(model_path)

def embed(text, prefix="query: "):
    inputs = tokenizer(f"{prefix}{text}", max_length=256,
                       padding=True, truncation=True, return_tensors="pt")
    with torch.no_grad():
        out = model(**inputs)
    emb = out.last_hidden_state.mean(dim=1)
    return F.normalize(emb, p=2, dim=1)

# Search
query = embed("iPhone 15", prefix="query: ")
product = embed("iPhone 15 Pro Max 256GB for sale", prefix="passage: ")
score = torch.cosine_similarity(query, product).item()
print(f"Similarity: {score:.3f}")
